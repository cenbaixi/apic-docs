## 规则弹窗:普通 / 高级 双模式(取消折叠)

同一个 form、两种显示模式,**提交始终读全部字段,普通编辑与高级编辑互不抹掉对方设置**:

**普通弹窗(创建 + 编辑)——只 4 行:**
1. 入口 · 接入层 + 监听端口
2. 出口 · 协议层(选出口分组后**仍跟着弹隧道协议**)
3. 落地地址 (IP/域名:端口)
4. PROXY protocol

**高级弹窗(高级编辑)——不含普通弹窗内容:**
转发方式 · 备注名 · SNI · 入口出站 · 入站绑定 · REALITY借用站点/TLS版本 · 出口出站 · 管理员JSON(及历史隐藏字段)。

- 取消了原折叠按钮;改为按打开方式切换:`编辑`=普通模式、`高级编辑`=高级模式。
- 字段联动(`applyRuleVis`)保留:隧道协议按"是否选出口"显隐,REALITY/TLS 借用站点按所选协议显隐。

## PROXY protocol → V1 / V2 / V3(对齐 nyanpass 语意)

下拉改为:关 / **V1(接收)** / **V2(发送)** / **V3(收发)**。
- V1 = 接收上游 PROXY 头;V2 = 向落地发送(v2 二进制头);V3 = 收 + 发。
- 后端字段(`proxy_recv` / `proxy_proto`)与提交/回填逻辑**未改**——选项值 `recv`/`v2`/`recv+v2` 正好对应。

## 设置页:系统卡拆分 + 新增「更新」卡
