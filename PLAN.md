## 一、三台机器分工(铁律:私钥永不上公网)

```
┌─ 临时云机(签码用,用完即焚)──────────────────┐
│  licensegen keygen / batch                       │
│  私钥只在此内存出现;签完码上传发卡网 → 销毁该机   │
│  公钥记下 → 进 master;私钥存 Bitwarden(见四)    │
└──────────────────────────────────────────────────┘

┌─ 发卡服务器(公网,块4/5)─────────────────────┐
│  已签好的"码池" + 下单 + 收款 + 发码 + 客户查询    │
│  TG bot 入口;付款回调 → 调 hub 设到期            │
│  ★ 碰不到私钥 ★(码是临时云机签好上传的)         │
└──────────────────────────────────────────────────┘

┌─ hub 机(公网,dl.apic.gg,块2 已上线)─────────┐
│  托管更新文件 + 心跳/有效期管理 + 一码一域名      │
│  hub.db(SQLite)存码注册表;管理 API 改期/吊销     │
│  ★ 碰不到私钥 ★;★ 务必备份 hub.db ★             │
└──────────────────────────────────────────────────┘

  客户服务器:跑 master(面板)。只有公钥验证,无私钥。
  apic(us-lax02):源码 + 编译 + GitHub 推送 + 开发验证。
```

## 二、授权体系(已定,最终形态)

```
① 授权码 = 离线预签"空白码":只签 {id, plan, hub:true},不绑域名/机器、不写到期。长码(复制粘贴)。
② 有效期 = hub 在线管理:售出时设、续费时顺延、心跳下发、master 本地缓存。
③ 绑定 = 首次激活绑域名(激活锁 + hub 一码一域名跨服务器强制),一码一域名,激活一次。
④ 续费 = 收款 → hub 改到期(extend 顺延)→ 心跳同步。私钥永不参与。
⑤ 暂停 = 本地已到期 且 连不上 hub 超 72 小时 才暂停;未到期时 hub 故障不影响(不误伤付费客户)。
⑥ 恢复 = 续费或 hub 恢复 → 心跳自动解除暂停。
⑦ 吊销 = hub 吊销名单/DB 即时拉黑(退款、盗用)。
⑧ 发卡 = 码池取码;域名/指纹下单时仅记录,真绑定在客户部署激活时。
```

**完整生命周期**:
```
你(临时云机)签 3000 空白码 → 上传发卡网码池
客户发卡网下单(选时长=商品)→ 付款 → 回调 → hub 给该码设到期(商品对应天数)
→ 发卡网发码给客户 → 客户 curl 一键装 → 填码+域名 → 激活(绑域名)
→ master 心跳 ↔ hub(下发到期)→ 正常运行
续费:客户再买 → 回调 → hub 顺延到期 → 心跳同步
到期未续 + 连不上 hub 超 72h → 暂停;续费后自动恢复
盗用/退款 → hub 吊销 → 即时拉黑
```

## 四、私钥保管(已定)

- **工具:Bitwarden**(免费版足够;高级 $10/年)。开源、云端加密、跨平台。
- **做法**:临时云机 `licensegen keygen` → 公钥记下(进 `cmd/master/license.go`)→ 私钥复制进 Bitwarden 的 Secure Note(强主密码 + 两步验证)→ 销毁云机。
- 以后签码:开临时云机 → 从 Bitwarden 取私钥粘进去 → `licensegen batch` → 上传码池 → 销毁云机。
- 私钥只在 Bitwarden(加密云端)+ 云机内存出现过,本地硬盘与公网服务器硬盘都不留。

## 五、生成 3000 码标准流程(待私钥就绪后执行)

```bash
# —— 在临时云机上(一次性)——
# 1. 首次:生成密钥对(若还没有)
./licensegen keygen          # 公钥→记下进 master;私钥→存 Bitwarden

# 2. 批量签 3000 个空白码(码统一,套餐 standard;时长靠发卡网商品区分,不进码)
./licensegen batch -priv <Bitwarden取出的私钥> -plan standard -count 3000 \
    -out codes.txt -ids code-ids.txt

# 3. codes.txt(3000 个码)→ 上传发卡网码池寄售
#    code-ids.txt(码ID清单)→ 留底(对账/排查用;hub 会在售出/心跳时自动登记)

# 4. 销毁这台临时云机
```
> 空白码不计时,放多久不坏;客户买了 hub 才设到期。

## 七、检测体系(★ 最后做,作为 update 测试内容 ★)

> 授权体系全部完成后再做。现仅存档需求,做时再核实内核现状。

**三端架构**:入口 → 跳板 → 节点(三跳转发)。

**检测方式**:tcping,每段 5 次。
- 单节点:对其出站目标 tcping 5 次。
- 群组 = **负载均衡(全连接)**:所有 A(入口)→ 所有 B(跳板),所有 B → C(节点,一般一个)。每段都测。

**频率与触发**:
- 后台每 10 分钟自动检测一次,存记录。
- 客户点"检测":调出**最近一次记录**(非当场重测)。

**故障判定与转移**:
- 检测中某入站节点的**出站 IP 不通(timeout)**→ 提示"该线路故障,临时舍弃"。
- **多出站 IP**(内核应支持,做时核实):若该入口配多出站 IP 且**设了自动切** → 自动切到通的 IP;**未设自动切** → 仅提示报错,不自动切。
- **恢复**:记录中某线路 timeout 后又恢复 → 提示**恢复时间 + 本次总丢失时长**。

**做前必须核实**:内核对"一入口多出站 IP + 自动切换"的真实支持程度(查 internal/kernel 源码)。
