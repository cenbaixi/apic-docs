# APIC / forward-panel 规则参数手册

规则的「高级 JSON」(advanced JSON)是一段 per-rule 的 JSON,填进规则编辑器的高级配置框,master 下发前用它覆盖/设置规则字段。本手册列**全部可配的 advanced JSON 键**。

> 不写的键 = 用默认。把需要的键合进同一个 JSON 即可。
>
> **最关键的键是 `relay_mode`**(中转外层伪装):它决定了壳整形(`shell_*`)、quic-obfs 整形(`quic_obfs_pad`)、TLS SNI(`tls_sni`)分别在哪种形态下才生效。先定 `relay_mode`,再配对应整形参数。

---

## 1. 中转外层伪装

「入站→出口跳板」这一段隧道的外层穿什么衣服。隧道**内层恒为 AES-AEAD**,这里只选外层伪装。

### `relay_mode` —— 外层传输(最重要)

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `""`/`aes`(裸 AES,无外层)· `shell`(自研壳)· `tls`(TLS 伪装)· `reality`(VLESS+REALITY 借真站)· `quic-obfs`(Salamander 混淆 QUIC) |
| 默认 | `aes` |
| 端口 | 出口跳板按档分端口:aes=base · shell=base+1 · tls=base+2 · reality=base+3 · quic-obfs=base+5(base=出口 relay 基准口,如 9100) |

**怎么选**:
- `aes`:同机房/可信链路,最快,无伪装开销。
- `shell`:自研壳,utls 真 Chrome 指纹 + 整形(见 §3)。通用跨墙。
- `tls` / `reality`:TLS 形态。`reality` 借真站证书,能对上 IP↔域名,最隐蔽但要配真站。
- `quic-obfs`:**唯一线上无明文 SNI** 的形态(整包加扰)。**最敌对的跨墙跳优先用它**;高丢包链路也优于 TCP。

> ⚠ 原生 `quic` 档已在 v1.8.0 移除。线上若还有 `relay_mode=quic` 的老规则,改成 `quic-obfs`。

### `tls_sni` —— TLS/quic-obfs 伪装 SNI

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 作用 | `relay_mode=tls` 的伪装 SNI;`quic-obfs` 的内层 QUIC TLS SNI |
| 默认 | 空 = `www.microsoft.com` |
| 说明 | 客户端用它作 ClientHello 的 ServerName;tls 档服务端按收到的 SNI 现签 CN 匹配的自签证书 |

### `decoy` —— 壳鉴权失败诱饵站 / REALITY dest

| 项 | 值 |
|---|---|
| 类型 | 字符串(`host` 或 `host:port`) |
| 作用 | ① 自研壳前置认证失败时回放到的诱饵站(空 = 伪装 SNI 的 :443);② `reality` 复用此字段为借证书的真站 dest |
| 默认 | 空 |
| 建议 | reality 档**必填**一个真实大站(如 `www.apple.com:443`);壳档可留空 |

### `tls_vers` —— Trojan 接受的 TLS 版本

| 项 | 值 |
|---|---|
| 类型 | 字符串(CSV) |
| 取值 | `1.3` · `1.2` · `1.2,1.3` |
| 作用 | 仅 Trojan-over-TLS 档(`encrypt=trojan`) |
| 默认 | 实现默认 |

## 2. 出口 IP 轮换 + 按量限量(route2 / #2)

对抗「单出口 IP 流量阈值封禁」。**出口 node 装机时要配多个 relay IP**(`-relay-addr 1.1.1.1:9100,2.2.2.2:9100,...`),否则下面的限量不触发(单 IP 没法轮换)。

### `exit_ip_cap_gb`

| 项 | 值 |
|---|---|
| 类型 | 整数(GB) |
| 作用 | 单个出口 relay IP 在窗口内的流量上限(**上下行合计、跨所有规则**) |
| 默认 | `0` = 不限量(仅轮换) |
| 语义 | 超帽的 IP 被踢出出口池;master 每 **60s** 重算一次(刚超帽踢出、降下来放回) |

### `exit_ip_cap_window_h`

| 项 | 值 |
|---|---|
| 类型 | 整数(小时) |
| 作用 | 滚动统计窗口 |
| 默认 | `48`(最大 168) |

### `exit_ip_cap_full`

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `""`/`rotate`(软限:全超帽时仍全部参与,保可用) · `stop`(硬断:全超帽时空池、拒绝出网) |
| 默认 | `rotate` |
| 注意 | `stop` 会直接拒绝出网(**不裸奔直连**),适合「宁可断不可裸」的强隐蔽场景 |

> **残留**:换 IP 解不了「IP↔ASN 深层不一致」(数据中心 IP 声称大站)。要彻底对上用 `reality`。

## 3. 自研壳整形(#3 / #4,仅 `relay_mode=shell`)

核心整形(填充量化 + 写入抖动 + keepalive)**默认开、无开关**;以下为可选项。tls/reality/quic-obfs 不受这些键影响。

### `shell_pad` —— 壳填充档

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `""`/`mtu`(激进:每帧填到 ~1.4KB,默认) · `256`(温和:在途向上取整 256B,省流量) |
| 默认 | `mtu` |
| 方向 | **仅入口→出口**(出口壳服务端跨规则共用,固定默认档;回程按默认) |

### `shell_session_max_s` —— 壳会话寿命

| 项 | 值 |
|---|---|
| 类型 | 整数(秒) |
| 作用 | >0 时入口在 ~该时长(随机化 `[半, 全]`)后主动关壳会话,促重连走新出口 IP |
| 默认 | `0` = 不限 |
| 副作用 | 长传输会被打断(客户端自动重连) |

### `shell_sni` —— 固定壳伪装 SNI

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 作用 | 固定壳每连接的伪装 SNI;空 = 每连接从内置 7 站池**轮换**(默认) |
| 默认 | 空(轮换) |
| 建议 | 设为**该出口 IP 实际能对上的站点**更隐蔽;否则留空轮换 |

> 壳 decoy(鉴权失败回放真站)默认开,无需配置;诱饵站见 §1 的 `decoy`。

## 4. quic-obfs 整形(#5,仅 `relay_mode=quic-obfs`)

### `quic_obfs_pad` —— datagram 填充档

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `""`/`256`(256 边界量化,默认) · `mtu`(填到 1280) |
| 默认 | `256` 边界 |
| 方向 | **仅入口→出口**(同 `shell_pad` 不对称;出口固定默认) |
| 上限 | 无论哪档,在途 UDP 报封顶 **1280B**(IPv6 最小 MTU,防 IP 分片) |

> ⚠ **部署**:v1.9.0 改了 quic-obfs wire 格式,**老↔新不兼容**。用 quic-obfs 的那一跳,入口+出口两端 agent 必须**同时**升级到 v1.9.0,升级窗口该模式短暂断流(其它模式不受影响)。噪声注入(入口侧 8–24s 抖动发噪声报)默认开、无需配置。

## 5. 接入层入站

### `in_bind` —— 入站监听绑定 IP

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 作用 | 入站监听绑定的本地 IP(传统转发 tcp 路径) |
| 默认 | 空 = 全部 `0.0.0.0` |

### `ingress_conn_max_s` —— 入站连接寿命(Tier1)

| 项 | 值 |
|---|---|
| 类型 | 整数(秒) |
| 作用 | >0 时入口在 ~该时长(随机化 `[半, 全]`)后主动关客户 TCP,稀释单入站 IP 的「超长连接」取证特征 |
| 默认 | `0` = 不限 |
| 覆盖 | forward 规则 + 单端口(SNI 路由)规则两条路径都生效 |
| 副作用 | 客户端会按此节奏重连;**建议 ≥300s**,太短频繁断流伤体验。上线前在 test2 确认你的客户端会自动重连 |

> 适用于**接入 IP 稀缺、不可轮换**时的力所能及保护(延缓取证,非根治)。

## 6. 源 IP 选择 + PROXY protocol

### `egress_policy` —— 入口出站多源 IP 选择

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `rr`(轮询,默认) · `random` |
| 作用 | 入口配了多个出站源 IP 时,在它们之间怎么选(配合规则的 `egress_ips`) |
| 默认 | `rr` |

### `exit_egress_policy` —— 出口→落地段多源 IP 选择

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | 同上(`rr` / `random`) |
| 作用 | **仅中转时**,出口机拨业务机的源 IP 选择(配合 `exit_egress_ips`,由 master 经 relayInner 下发给出口) |
| 默认 | `rr` |

### `proxy_proto` —— PROXY protocol 发送

| 项 | 值 |
|---|---|
| 类型 | 字符串 |
| 取值 | `""`(关) · `v1`(文本头) · `v2`(二进制头) |
| 作用 | 开启后入站把 PROXY 头作为首段写入上游(经中转透明到达落地),**让落地拿到真实客户端 IP** |
| 默认 | 关 |

### `proxy_recv` —— PROXY protocol 接收

| 项 | 值 |
|---|---|
| 类型 | 布尔 |
| 作用 | 解析客户端/上游传入的 PROXY 头(v1/v2),取其中原始源地址。**链式中转**用:前一跳已带 PROXY 头时,把最原始客户端 IP(而非前一跳 IP)继续传给落地 |
| 默认 | `false` |

## 7. 全参数速查表

```json
{
  "relay_mode": "shell",
  "tls_sni": "www.microsoft.com",
  "decoy": "www.apple.com:443",
  "tls_vers": "1.3",

  "exit_ip_cap_gb": 100,
  "exit_ip_cap_window_h": 48,
  "exit_ip_cap_full": "rotate",

  "shell_pad": "256",
  "shell_session_max_s": 1800,
  "shell_sni": "",

  "quic_obfs_pad": "mtu",

  "in_bind": "",
  "ingress_conn_max_s": 600,

  "egress_policy": "rr",
  "exit_egress_policy": "rr",
  "proxy_proto": "v2",
  "proxy_recv": false
}
```

| 键 | 类型 | 作用形态 | 方向 | 默认 |
|---|---|---|---|---|
| `relay_mode` | str | 全 | 中转外层 | `aes` |
| `tls_sni` | str | tls / quic-obfs | 中转 | `www.microsoft.com` |
| `decoy` | str | shell / reality | 中转 | 空 |
| `tls_vers` | str | trojan | —— | 实现默认 |
| `exit_ip_cap_gb` | int | 全(出口多 IP 才生效) | 出口 | 不限 |
| `exit_ip_cap_window_h` | int | 同上 | 出口 | 48h |
| `exit_ip_cap_full` | str | 同上 | 出口 | `rotate` |
| `shell_pad` | str | shell | 入口→出口 | `mtu` |
| `shell_session_max_s` | int | shell | 会话级 | 不限 |
| `shell_sni` | str | shell | —— | 轮换 |
| `quic_obfs_pad` | str | quic-obfs | 入口→出口 | `256` 边界 |
| `in_bind` | str | 全(tcp 路径) | 接入层 | `0.0.0.0` |
| `ingress_conn_max_s` | int | 全 | 接入层入站 | 不限 |
| `egress_policy` | str | 全(入口多源 IP) | 入口出站 | `rr` |
| `exit_egress_policy` | str | 中转(出口多源 IP) | 出口出站 | `rr` |
| `proxy_proto` | str | 全 | 入站→上游 | 关 |
| `proxy_recv` | bool | 全 | 上游→入站 | false |

## 8. 按场景推荐配置

**A · 通用跨墙(壳)**
```json
{ "relay_mode": "shell", "shell_pad": "256", "shell_session_max_s": 1800 }
```

**B · 最敌对跨墙跳(无 SNI)**
```json
{ "relay_mode": "quic-obfs", "quic_obfs_pad": "mtu" }
```
(出口+入口两端都要 v1.9.0)

**C · 出口 IP 抗封(多 IP 出口)**
```json
{ "relay_mode": "shell", "exit_ip_cap_gb": 100, "exit_ip_cap_window_h": 48, "exit_ip_cap_full": "rotate" }
```
(出口装机 `-relay-addr` 配逗号多 IP)

**D · 入站 IP 稀缺、想稀释取证**
```json
{ "relay_mode": "shell", "ingress_conn_max_s": 600 }
```

**E · 让落地拿真实客户端 IP**
```json
{ "relay_mode": "aes", "proxy_proto": "v2" }
```

**F · 最隐蔽(借真站,IP↔域名对得上)**
```json
{ "relay_mode": "reality", "decoy": "www.apple.com:443" }
```
