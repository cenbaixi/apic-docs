# 高级参数（JSON）

除了规则编辑器里的常规字段，部分**进阶能力**通过规则的「高级配置」JSON 开启——主要是**出口 IP 抗封、链路整形、入站连接保护**这几类抗封锁加固。本页逐项说明。

!!! tip "在哪里填"
    在规则编辑器的**高级配置 / 高级 JSON** 输入框里填一段 JSON。**不写的键 = 用默认**，把需要的键合进同一段 JSON 即可。常规字段（伪装档、PROXY protocol、出站源 IP 等）见 [参考 → 规则字段](../reference.md#规则字段)，本页只讲高级 JSON 项。

---

## 出口 IP 轮换 + 限量

对抗「单个出口 IP 跑量太大被封」。给出口挂多个 IP，面板按流量自动把跑超的 IP 轮换下去。

!!! warning "前提：出口节点要装多个 IP"
    限量只在**出口节点配了多个中转 IP** 时才生效（单 IP 没法轮换）。部署出口节点时，安装命令的中转地址填**逗号分隔的多个**，例如 `1.1.1.1:9100,2.2.2.2:9100,3.3.3.3:9100`。只配一个 IP 时这些参数无效。

```json
{
  "exit_ip_cap_gb": 100,
  "exit_ip_cap_window_h": 48,
  "exit_ip_cap_full": "rotate"
}
```

| 键 | 含义 | 默认 |
| --- | --- | --- |
| `exit_ip_cap_gb` | 单个出口 IP 在统计窗口内的流量上限（GB，**上下行合计、跨所有规则**）。超上限的 IP 被轮换下去 | `0`＝不限量 |
| `exit_ip_cap_window_h` | 流量统计的滚动窗口（小时），最大 168 | `48` |
| `exit_ip_cap_full` | 池里 IP **全部**超上限时怎么办：`rotate`＝照常轮换、保持可用；`stop`＝暂停该规则、拒绝出网 | `rotate` |

- 面板每分钟重算一次：刚跑超的 IP 踢出、流量降下来的放回，**无需手动干预**。
- `stop` 会在全部超限时**直接拒绝出网**（不会改走裸连接），适合「宁可断也不暴露」的场景；一般用 `rotate`。

!!! note "解决不了什么"
    轮换 IP 缓解的是「流量阈值封 IP」。它**不能**改变「机房 IP 却声称大站」这种 IP 与域名对不上的深层特征——那需要用 **REALITY** 档（借真实站点证书）。最敌对的线路也可优先用 **QUIC + 混淆**（链路上没有明文域名）。

## 自研壳整形

仅作用于伪装档 = **自研壳**。壳的核心整形（填充、写入抖动、保活）**默认就开、无需配置**；以下是可选微调。其它伪装档（TLS / REALITY / QUIC+混淆）不受这些键影响。

```json
{
  "shell_pad": "256",
  "shell_session_max_s": 1800,
  "shell_sni": ""
}
```

| 键 | 含义 | 默认 |
| --- | --- | --- |
| `shell_pad` | 填充力度：`mtu`＝每包填到约 1.4KB（更均一）；`256`＝按 256 字节量化（更省流量） | `mtu` |
| `shell_session_max_s` | 壳会话最大寿命（秒）。设置后到点（带随机）主动断开重连，配合上面的出口 IP 轮换可周期性换出口 IP。`0`＝不限 | `0` |
| `shell_sni` | 固定壳每条连接伪装的站点域名；**留空＝每条连接自动轮换**（默认，更隐蔽）。若设，建议填**该出口 IP 实际能对上的站点** | 空（轮换） |

!!! note "只作用单向"
    `shell_pad` 只影响**入口 → 出口**方向的填充；回程按默认。这是设计取舍，不影响使用。`shell_session_max_s` 会打断长连接（客户端自动重连），按需开启。

## QUIC + 混淆整形

仅作用于伪装档 = **QUIC + 混淆**。在已有的整包加扰之上，把 UDP 包大小量化、并注入噪声包，进一步抹平流量形状。**噪声注入默认开、无需配置**。

```json
{ "quic_obfs_pad": "mtu" }
```

| 键 | 含义 | 默认 |
| --- | --- | --- |
| `quic_obfs_pad` | UDP 包填充：`256`＝按 256 字节量化；`mtu`＝填到 1280（更均一、更费流量）。无论哪档都**封顶 1280 字节**以避免分片 | `256` |

!!! warning "升级须两端同时进行"
    此能力调整了 QUIC + 混淆的链路格式，**新旧不兼容**。用了「QUIC + 混淆」档的那一跳，**入口和出口两端节点必须同时更新**到支持的版本，升级窗口内该档短暂断流（其它伪装档不受影响）。升级完两端再确认连通。

## 接入层入站连接寿命

面向**接入层 IP 紧张、无法轮换**的场景：到点主动断开客户端连接、促其重连，**稀释单个入站 IP 上「超长连接」的取证特征**。对传统转发与单端口 SNI 两种规则都生效。

```json
{ "ingress_conn_max_s": 600 }
```

| 键 | 含义 | 默认 |
| --- | --- | --- |
| `ingress_conn_max_s` | 客户端入站连接最大寿命（秒）。到点（带随机）主动断开，客户端会自动重连。`0`＝不限 | `0` |

!!! note "尽力而为，不是根治"
    入站 IP 没有更换，这只是**延缓取证、增加判定难度**。**建议 ≥300 秒**，太短会频繁断流影响体验。上线前在测试线路确认你的客户端协议能正常自动重连。

## 速查表

把需要的键合进同一段 JSON：

```json
{
  "exit_ip_cap_gb": 100,
  "exit_ip_cap_window_h": 48,
  "exit_ip_cap_full": "rotate",
  "shell_pad": "256",
  "shell_session_max_s": 1800,
  "shell_sni": "",
  "quic_obfs_pad": "mtu",
  "ingress_conn_max_s": 600
}
```

| 键 | 作用伪装档 | 默认 |
| --- | --- | --- |
| `exit_ip_cap_gb` / `exit_ip_cap_window_h` / `exit_ip_cap_full` | 全部（出口需配多 IP） | 不限量 |
| `shell_pad` / `shell_session_max_s` / `shell_sni` | 自研壳 | 见各项 |
| `quic_obfs_pad` | QUIC + 混淆 | 256 量化 |
| `ingress_conn_max_s` | 全部 | 不限 |

!!! tip "怎么选组合"
    - 通用抗识别：`自研壳` + `shell_pad: "256"`
    - 最敌对线路（无明文域名）：`QUIC + 混淆` + `quic_obfs_pad: "mtu"`（两端同时升级）
    - 出口 IP 抗封：任意档 + `exit_ip_cap_*`（出口配多 IP）
    - 接入 IP 紧张：任意档 + `ingress_conn_max_s`
    - 最隐蔽（IP 与域名对得上）：`REALITY` + 借真站
