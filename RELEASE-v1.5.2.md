# forward-panel v1.5.2 — 公告即时下发修复 + 分组节点调整改单按钮弹窗

两个问题。涉及 hub + master + 前端。agent 不变。

## 问题 2(更要紧)· 公告发布后要几小时才显示 → 改为实时拉取

**现象**:在 hub 发布公告后,分销面板「设置-更新 → 更新日志」迟迟不显示。

**真因**:不是没接通(v1.5.0 链路是对的),而是公告**搭了授权心跳的便车**,而授权心跳间隔是 `6 * time.Hour`(正常态 6 小时一次 + 随机)。所以面板最长要 6 个多小时才在下次心跳拉到公告。

**显示位置(回答"在哪显示")**:面板 **设置 → 更新 → 更新日志**(`#clList`),数据来自 `/api/changelog`。

**修复(实时化,不动授权心跳)**:
- **hub**:新增公开端点 `GET /changelog`,返回最新 N 条公告数组(无鉴权——公告本就是给分销看的)。
- **master**:`GET /api/changelog` 改为**优先实时从 hub 拉**(`/changelog`,3s 超时),成功则刷新本地缓存并返回;失败/超时**回落**本地缓存(心跳下发的 `changelog_json`)。
- 于是管理员一打开「设置-更新」就实时看到最新公告;hub 不可达时仍显示上次缓存。

## 问题 1 · 分组节点调整:每节点一个按钮 → 整列一个「查看」按钮 + 弹窗批量调

原「分组」页节点列里**每个节点**一个查看按钮。改为:
- 整列**只一个「查看」按钮**(旁边仍列出本组节点名);
- 点击弹出 `groupNodesModal`,**列出本组所有节点**,每个节点一行带分组下拉(显示节点名/ID/层);
- 「保存」**批量提交**所有改动的节点(只提交改了的),完成后刷新(顺序 loadNodes→loadGroups)。
- 删除了上版遗留的单节点弹窗与不再使用的 `moveNodeGroup`。
