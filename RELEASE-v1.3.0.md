# forward-panel v1.3.0

## 修复 / 优化
- **带宽残留尖峰**:`sampleTrafficOnce` 跳过离线节点——离线期不采样,节点回上线后那段流量按真实大间隔均摊成低速率,不再塞进一个 300s 桶冒成单点尖峰(配合 v1.2 的 >50GB/s 钳制 + prev≤0 归零)。
- **节点 ID 自动生成**:面板添加节点时 ID 留空 = 后端自动生成 `n-{8hex}`,输入框在添加界面隐藏(编辑时仍只读显示作标识)。后端 apiNodeMeta 回传生成的 ID 供 provision/SSH 用。
- **规则列表隐藏 rule-xxx**:规则有备注名时只显名字,隐藏 `rule-xxxxxx` 串(无名规则才回落显示 ID)。
- **入口公网 IP 显示**:节点列表优先显示 master 观测到的连接远端地址(真公网),NAT 机不再显自报的私网 `ips[0]`。
- **分段检测(入口侧)**:入口节点探测其实际拨号的下一跳——有跳板则探第一跳(跳板入口),否则探最终落地。修正了"入口直接探隔着跳板、根本够不到的落地"导致的误判。

## 暂挂(后续聚焦版本)
- **分段检测·完整多段**:出口/中转节点探各自下一跳,需 master 按拓扑把每节点下一跳目标推下去(新协议消息)。只影响探测路径、不碰转发数据路径,单独迭代 + 隔离测试。
