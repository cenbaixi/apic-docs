# forward-panel v1.4.4 — 监听口未绑上上报 + 前端修补(含 v1.4.3 分级检测)

本版 = v1.4.3(中转分级检测 + 真实传输端口探活,见 RELEASE-v1.4.3.md)+ 以下新增。

## 新增:监听口未绑上(bind 失败)上报
入口规则的监听口若没绑上(端口冲突 / 被占用等),心跳本不报、入口→隧道探测也不报(隧道跟监听口无关),过去只在 agent 日志里。现在:

- **agent**:`k.Apply` 失败时记下该规则的 bind 错误(`ruleID → err`),随心跳上报 master;Apply 成功 / 规则移除时清掉。
- **master**:存于节点,`apiRuleSegments` 逐入口收集,响应新增 `listen_bind` 字段。
- **前端**:检测弹窗顶部出红条幅「⚠ 监听口未绑上 · 入口X 的 :PORT 没起来(端口冲突/被占用)」并附原始错误。

## 前端修补(纯 web,无 Go 改动)
- **批量勾选丢失**:规则表每秒轮询重渲染会冲掉复选框。改为持久选中集 `RULE_SEL`,跨重渲染保持勾选,全选带半选态,删除规则自动清残留。`selectedRuleIds()` 照常工作。
- 检测弹窗说明文字由旧的「入口段(规则监听口)」改为「入口→出口中转口 + 出口→落地」,与 v1.4.3 分级段一致。

## ⚠ 部署:本版需同时更新 agent
v1.4.3 是 master 单边即可;**v1.4.4 的 bind 上报需要新 agent 发 `BindErrs`、新 master 读**。proto 字段是 `omitempty` 向后兼容(旧 agent 不报、不报错),但 **proto 版本号已 bump 0.87 → 0.88**,旧 agent 会显示「需更新」提示你升级。

部署顺序(test2 先验):
1. us-lax02 `./release.sh 1.4.4` 编译 master + agent。
2. test2:更新 master + agent(部署脚本本就同时拷两个二进制),验证:
   - 规则检测分级段正确(v1.4.3 行为);
   - 故意让某入口规则端口冲突(占用其 listen 口)→ 检测弹窗顶部出现「监听口未绑上」红条幅;清除冲突后几秒消失;
   - 批量勾选不再每秒丢失。
3. 生产:更新 master,再逐节点更新 agent(更新前节点显示「需更新」,更新后消失)。

bind 上报对旧 agent 静默降级(只是不显示该红条幅),不影响转发与其余检测。
