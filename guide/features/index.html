    功能说明¶

    除了端口转发与中转，APIC 面板还内置了若干运营功能。本页逐个说明。

    转发与中转¶

    面板的核心能力，详见 协议说明。

    • 多协议入站：传统 TCP + UDP 端口转发、单端口入 SNI 分流、自研壳、Trojan、TUIC、VLESS+REALITY。
    • 链路中转：入口 → 出口两段，中转隧道可选裸 AES、自研壳、TLS、REALITY、quic-obfs 等外层伪装。
    • 链式多跳：多个协议层节点串联，逐跳转发；配合 PROXY protocol 可把最原始客户端 IP 一路传到落地。

    线路 · 分组 · 计费¶

    • 线路（分组）：把同类节点编入「线路」，带「层 / 等级 / 倍率」。用户在规则里按线路选入口或出口，而非逐台选机器。
    • 倍率：线路内所有节点的流量按此倍率折算计入用量（贵的落地设高倍率）。
    • 用户等级：用户等级 ≥ 线路等级才可选用该线路。
    • 流量计费：按字节计量，入口 / 出口分别按倍率折算「已消耗流量」，口径与同类面板（nyanpass）一致，便于迁移。

    出口 IP 池轮换 + 单 IP 限量¶

    对抗「单出口 IP 流量阈值封禁」。

    • 出口节点装机配多个 relay IP（-relay-addr 逗号分隔），面板把它们组成出口池。
    • 每条连接在池内轮换出口 IP（壳 seed 按节点而非按 IP，轮换不破握手）。
    • 配 exit_ip_cap_gb / window_h / full 后，面板按「滚动窗口内单 IP 总流量」过滤超帽 IP，把未超帽的加权进池；stop 档全超帽时直接拒绝出网（不裸奔）。

    配置见 参数说明 · 出口 IP 轮换。

    接入层无感重连¶

    接入 IP 稀缺、不可轮换时，稀释单入站 IP 的「超长连接」取证特征。

    • 开启后入口在设定时长后主动关客户连接，但走「无感」收尾：等连接空闲再以 FIN 优雅断开，客户端自然重连，不腰斩传输。
    • 在规则高级编辑里勾选「启用主动中断」即可，参数见 高级编辑逐项。

    DNS¶

    面板内置两套 DNS 能力，面向「让落地解析走自己的链路」与「给客户分发可控解析」。

    split-horizon DNS（节点内置）¶

    每个节点的 agent 自带一个 split-horizon DNS 监听（默认 UDP+TCP :5353），可按规则对特定域名给出指定解析，落地侧的域名解析也能走可控路径，避免落地机直接用公共 DNS 暴露真实查询。

    聚合 DNS（面板管理）¶

    面板提供聚合 DNS 管理，把多个 DNS 厂商的记录统一在面板里 CRUD：

    • 厂商配置：在面板配置各 DNS 服务商的接入信息。
    • 记录管理：统一增删改查解析记录，一处管理多家。
    • 解析放行：配置允许的解析项。

    面板入口

    DNS 相关功能在面板有独立页签，分「记录」「厂商」「放行」几部分。具体接入哪些厂商以面板当前版本为准。

    CDN（边缘加速）¶

    面板内置 GoEdge 式的边缘 CDN 能力，给站点主提供加速 + 自查流量。

    • 边缘节点：把节点作为边缘，回源到源站，缓存命中直接由边缘返回。
    • 源站探活：边缘持续探活各源站，随心跳上报健康状态。
    • 聚合统计：每个站点的请求数 / 命中 / 未命中 / 字节，随心跳上报（刻意不含任何客户端 IP / 明细）。
    • 访问日志：站点主自查流量用的边缘访问日志（IP / 国家 / UA / Referer / 协议全记录），供站点主排查。

    隐私边界

    聚合统计与访问日志是两套独立数据：聚合统计不含客户端 IP，只是计数；访问日志才含明细，供站点主自查。

    节点管理¶

    • 状态总览：节点在线状态、CPU / 内存 / 负载 / 上行下行、IPv6 能力、连接数（TCP / UDP 分列）、协议版本一目了然。
    • tcping 探测：节点对落地目标做 tcping，给出 RTT / 丢包 / 连续失败次数，规则列表里直接看链路质量。
    • 节点装机 / 离线包：面板生成装机命令；无公网拉取环境可用离线包。
    • Web 终端：面板内直接对节点开终端（建议公网部署时用 -pty-allow 设白名单限制）。

    集中更新与公告¶

    • 一键检查更新：面板向更新中心（hub）检查新版本，签名校验后应用。
    • 更新日志 / 公告：由官方统一下发，面板内查看。

    更新后需重启生效

    应用更新会把新版拉到磁盘，但面板进程需重启才加载新前端。更新后请按提示重启面板服务。

    通知（Telegram）¶

    面板支持 Telegram Bot 通知，可配代理 URL 以适配中国大陆网络。在面板「设置 → Telegram」配置 Bot 与可发测试消息。
